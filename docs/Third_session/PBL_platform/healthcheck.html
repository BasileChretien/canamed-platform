CaNaMED platform health check

Bookmark this page and open it ~30 minutes before each live session. Each row below is a quick check. Green is good. Red needs attention before you start handing out the session code — see OPERATOR_INCIDENT_RUNBOOK.md for first-line fixes.

  1. HTTPS load + brand mark renders If you can see this page, the host is up and the bundle was served end-to-end.
    …
  2. localStorage write / read / delete Used for session unlock + theme + language + telemetry buffer.
    …
  3. sessionStorage write / read / delete Holds the per-tab clientId. Required for presence semantics.
    …
  4. Firebase config loaded window.CANAMED_FIREBASE must be set for shared mode. Empty = local-test only.
    …
  5. Firebase Realtime DB reachable Pings /.info/connected with a 5 second timeout. Red here usually means a network or DB outage.
    …
  6. App Check provider available Checks that window.CANAMED_RECAPTCHA_SITE_KEY is set and the reCAPTCHA script is reachable.
    …

This page does not require a session password — it is intentionally open so any facilitator can run it pre-flight from any device.
