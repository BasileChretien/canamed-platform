2. データ収集の利用目的

あなたのデータは、以下の利用目的にのみ使用します。新たな利用 目的が生じた場合は、新たな同意通知をお示しします。

3. 法的根拠

GDPRに基づく根拠：

  • 第6条第1項(a) — あなたの明示的な同意。 参加登録フォームの2つのチェックボックスにより取得します。 同意はいつでも撤回できます（§10参照）。
  • 第9条第2項(a) — 自由記述の回答中に開示される可能性のある 要配慮個人情報（特別カテゴリーのデータ） （健康関連情報、オピオイド使用・終末期・家族への告知等に 関する宗教的・哲学的見解など）については、当該カテゴリーに 対する明示的な同意を別途根拠とします。
  • 研究プロジェクト（上記の利用目的3）にデータを使用する 場合は、第9条第2項(j) — 科学研究を根拠 とし、第89条の保護措置（仮名化、アクセス制限、個別の決定を 行わないこと）を講じます。

APPIに基づく根拠：

  • 第17条／第21条 — 取得前に上記のとおり利用目的を特定。
  • 第20条第2項 — 要配慮個人情報に関する 事前のオプトイン同意。同じ同意チェックボックス により取得します。
  • 第27条第5項 — §1記載のとおり共同利用。

日本の参加者にとっては、APPI第28条に基づく 越境移転に該当します。日本の個人情報保護委員会（PPC）は、 2019年のEU・日本相互十分性認定により、欧州連合を日本と同等の 保護水準を有する地域として指定しています。当該移転に関する記録は、 APPI第28条第3項に基づき内部で保管しています。

Googleでサインインした場合、IDトークンに関する通信は米国に所在 するGoogle LLCの事業体によっても処理されます。これは、EUの標準 契約条項およびGoogleが公表する技術的・組織的措置の下で行われます。 当該措置を参照する移転影響評価（TIA）の記録を保管しています。

8. 保有期間

  • ライブセッションのデータ （sessions/{code}/pool内のお名前、在室状況、 入力中インジケータ）は、ワークショップ終了後 7日以内にライブデータベースから削除します。
  • セッションアーカイブJSON（セッション終了時に ファシリテーターがダウンロードするもの）は、30日以内に 仮名化されます — 名前はコードに置き換えられます — そして 対応表は6か月以内に廃棄されます。識別可能なコピーも同時期に 廃棄します。
  • 研究利用に関する2番目の同意ボックスにチェックを入れた場合、 仮名化されたデータセットは、関連する公表から 最長5年間保持され、その後不可逆的に削除されます。
  • Googleでサインインした場合、users/{uid}配下の プロフィールおよび履歴は、「マイアカウント」 パネルから削除されるまで保持されます。

9. あなたの権利

あなたには以下の権利があります。

  • 当方が保有するあなたの個人データへの開示・アクセス （GDPR第15条、APPI第33条）。
  • 不正確なデータの訂正（第16条／第34条）。
  • データの消去（第17条／第35条）。
  • 処理の利用停止・制限（第18条）。
  • 可搬性のある形式でのデータ受領（第20条）。
  • 処理に対する異議申立て（第21条）。
  • 将来に向けた効果をもって、いつでも同意を撤回する権利 （第7条第3項；APPI第38条第3項）。

これらの権利のいずれかを行使される場合は、 canamed-ethics@unicaen.fr (データ主体の権利行使のための安定した連絡先メールアドレス。 共同管理者により監視されています) までご連絡ください。 請求の受領は5営業日以内に確認し、 30日以内に回答します （GDPR第12条第3項；APPI第38条）。

権利の行使は、あなたの成績、大学における立場、ワークショップへの 参加に対していかなる影響も及ぼしません。 拒否または撤回によって不利益を被ることは一切ありません。

10. 同意の撤回／ワークショップからの退出

研究目的の利用に関する同意は、いつでも撤回できます。ワーク ショップ後は、§9に記載のメールアドレスに、セッションコードと お名前を添えてご連絡ください — 仮名化された研究アーカイブから あなたの投稿内容を削除し、書面で確認いたします。撤回は、既に 公表された分析に使用されたデータについては、これを撤回不能な 場合に遡及的影響を及ぼすものではありませんが、それ以降の分析に あなたのデータが含まれることはありません。

ワークショップ中は、いつでも退出ボタンを押して退出 できます。入力中のバッファおよび在室情報は直ちに削除されます。 すでに送信済みの投稿内容は、上記の連絡先に削除を別途請求しない 限り、ファシリテーターのデブリーフィング用コピーのためにルームの 記録に残ります。

12. 自動化された意思決定

本プラットフォームは、チームが入力した回答の内容、および チームが行った臨床選択（キーワードによるスコアリングおよび ペナルティ一覧）に基づき、自動的に得点を加算または減点します。 これらのスコアはセッション内の教育目的にのみ 用いられ、大学の成績には反映されず、あなたの記録に影響を 与えることもありません。スコアに異議がある場合は、セッション中に ファシリテーターに申し出ていただければ、人による確認を受ける ことができます。

プロファイリングや、あなたに対して法的効果またはそれに類する 重大な影響を及ぼす自動化された意思決定は、いずれも行いません （GDPR第22条；APPI第26条の3）。

13. セキュリティ

個人データはHTTPSで送信され、同じセッションの参加者のみが読み 書きできるようアクセス制御ルールで保護されたFirebase Realtime Databaseに保管されます。セッションの特権フィールド（管理者 パスワードのハッシュ、シナリオ、セッション作成マーカー）は、 セッションが作成された時点で不変となります。セッションパスワード は、PBKDF2-SHA256のハッシュ（10万回の反復、セッションコードを ソルトとする）として保存されます。自動デプロイに使用される サービスアカウントキーは、GitHub Actionsのシークレットとして 保管され、Firebase HostingおよびRealtime Databaseの管理権限 にのみスコープが限定されています。

これらの措置にもかかわらず、プラットフォームの侵害は起こり 得ます。あなたのデータに影響を及ぼす侵害が発生した場合：

  • GDPR第33条・第34条に基づき、管理者は 72時間以内にCNILへ通知し、あなたの権利に対する高いリスクが 生じる可能性がある場合は、あなたにも直接通知します。
  • APPI第26条（2022年改正）に基づき、運営者は 3〜5日以内にPPCへ速報を、30日以内に確報を提出し、影響を受ける 各データ主体への通知を行います。

17. お使いの端末上のローカルデータ

ブラウザは、プラットフォームの動作に必要な小さなデータをお使いの 端末に保存します。

  • sessionStorage：タブごとにランダムに生成 されるクライアント識別子（canamed_client）。 タブを閉じると消去されます。
  • localStorage：解錠済みのセッションコード （canamed_session）、自動復帰用のお名前および 参加情報（canamed_resume）、お祝いサウンドの設定 （canamed_sound）、選択されたUI言語 （canamed_lang）、そして — ファシリテーター用 のみ — 直近のワークショップ設定をワンクリックで複製できる ようにするためのデータ（canamed_last_workshop： ラベル、シナリオID、ルーム数、Teams／アンケートのリンク — パスワードは一切含みません）。これらの多くは、 退出ボタンまたはセッション終了画面の CANAMEDに戻るボタンを押した時に消去されます。 言語設定および複製用スタッシュは、ご自身で消去するまで残ります。
  • sessionStorage（ブラウザタブごと、タブを 閉じると消去）：未捕捉のJavaScript例外およびコンテンツ セキュリティポリシー（CSP）違反を最大50件記録する、小さな ページ内エラーログバッファ。このバッファが ブラウザの外に出ることはありません。バグ報告をいただいた際に、 ファシリテーターが管理ダッシュボードのエラーログを ダウンロードボタンから手動でエクスポートすることが できます。それ以外の場合は、タブを閉じると破棄されます。

本プラットフォームは、サードパーティCookie、トラッキング ピクセル、広告識別子のいずれも読み込みません。

ページから読み込まれる第三者サービス

プラットフォームの動作のため、ブラウザは少数の第三者スクリプト およびリソースも読み込みます。これらのいずれも、お名前、所属 大学、自由記述の回答、その他あなたが入力したいかなる内容も 受け取りません。

  • Google Firebase SDK（認証、リアルタイム データベース、App Check、パフォーマンス）— www.gstatic.com から、SubResource Integrityハッシュ付きで読み込まれます。 これにより、CDNが侵害された場合でも差し替えはできません。 これらは、Googleサインインを使う場合のサインイン、および 同じルームの他の参加者とのライブワークショップ状態の同期を 可能にするものです。
  • reCAPTCHA v3（Classic） — Googleのボット 検出サービス。Firebase App Checkを介してwww.google.com から、クラシック（無料枠）のv3プロバイダで読み込まれます。 受け取るのは、あなたが本物のユーザーで あることをリクエストごとに証明するトークン要求のみ であり、お名前、所属大学、回答、ワークショップの内容は一切 参照されません。reCAPTCHAに関するGoogleのデータ取扱いは、 Googleプライバシーポリシーに従います。
  • Firebase Performance Monitoring — 運用者 がデプロイ時に有効化した場合（firebase-config.js の CANAMED_PERF_MONITORING を参照）、Firebase SDK は匿名のタイミングデータを Google に送信します：ページ 読み込み時間、First Contentful Paint、Time to First Byte、 およびページが発行する各ネットワークリクエストの所要時間と ステータスコードです。参加者が入力したテキスト、お名前、 所属大学、同意の値、セッションコード、clientId は一切 送信されません。このデータはリアルユーザーのパフォーマンス モニタリング（例：コホート間でプラットフォームの応答性を比較する） に使用されます。取扱いは Firebase プライバシーおよび セキュリティのページおよび Googleプライバシーポリシーに準拠します。
  • QRコードの生成 — 同梱のqrcode.js により、すべてクライアント側で生成されます。外部リクエストは 一切発生しません。QRコードに含まれるのは、公開されている セッション参加用URLのみです。
